先前專案並沒有具體規範應該如何定義 UI 當中的數值，導致魔法數字（沒有規範與描述的值）流竄於整個產品當中，造成了非常大的困擾。

因此這次架構翻新查閱了一下 Figma 關於 Design Token 相關文件並總結出新文章，引導團隊了解可以快速了解如何管理數位產品當中的 UI 變數，透過以下三種分類方式：

- 原始值變數 Primitive Variable
- 語意化變數 Semantic Variable
- 元件變數 Component Variable

詳細可以參考：[具體來說可以怎麼建立與管理產品的設計令牌或變數](https://www.webdong.dev/post/how-to-organize-your-digital-product/)
