如果有個很慢的網站只能挑一種方式去改善，那我一定會從網站內容本身下手：圖片、影片、字體、音訊，因為這是最低投入但最高回報的選項，你可以……

- 考量用戶需要多精細的資料，只載入必要內容（懶加載、適當尺寸影像）
- 考量用戶環境，採用更有效率更貼切場合需求的格式（gif / webp）（SVG / PNG）

大多時候這些不需要實際碰到任何開發細節的部分才是構成網頁體驗差勁的元兇。

- 以上是網頁傳統文件性質的一面，網頁從原先文件性質更多的朝向應用程式體驗發展，那就真的是開發需要努力的領域了，這裡有個有趣的統計：[JavaScript Bloat in 2024](https://tonsky.me/blog/js-bloat/)。
- 就算是簡單的圖片也不容小覷，之前寫了一篇大長文細挖網頁圖片的奧妙：[所有最佳化網頁圖片的技巧都寫在這了！](https://www.webdong.dev/post/all-you-need-to-know-about-optimal-images/)。
