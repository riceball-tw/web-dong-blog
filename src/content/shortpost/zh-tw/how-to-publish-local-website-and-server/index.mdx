先看這兩個 Meme 怎麼解：

👉 誤以為其他電腦可以直接連接本地檔案

善用靜態網頁托管服務，上傳到 GitHub Pages、Netlify 或 Vercel ……等服務生成一個公共 URL，其他人便能透過該連結來查看網站。

👉 誤以為其他電腦可以直接連接本地伺服器端口

使用內網 IP 地址：在同一個內部網路（例如在同一個 Wi-Fi）下，可以使用本地 IP 位址來訪問伺服器，這樣其他同網段的裝置可以直接通過這個 IP 位址進行訪問。
公開本地伺服器：如果希望在外網訪問本地伺服器，可以使用一些工具如 ngrok、LocalTunnel 或者 Cloudflare Tunnel 創建一個暫時網址，將本地伺服器映射到外網。

前端時常要在不同裝置與瀏覽器環境上測試，而常用的 [VSCode 編輯器其實就有內建 Microsoft dev tunnels](https://code.visualstudio.com/docs/editor/port-forwarding)，可以迅速幾個按鈕把開發伺服器放上網測試。

甚至 [Nuxt 有內建支援 Cloudflare Quick Tunnels](https://www.youtube.com/watch?v=b4nXOFTXI_c)
