身為前端工程近期工作的感悟是很多時候開發問題還是陷於介面的外觀、行為互動層面上，不經讓我感嘆客製化 UI 的開發與維護成本對小團隊來說實在是太高了。

一方面希望「快速推出產品」而另一方面又希望有「高品質的實踐成果」，結果就是做正確的事一直都很困難。

正確的事包含：效能、適用性、可維護性、可測試性、搜尋引擎最佳化、跨平台相容性、多語系……思考並列出了一些現有團隊遇到的問題，並且最重要的可以怎麼面對這些問題的結論：- 保持網頁 UI 的單純樸素

- 尊重瀏覽器標準
- 善用現成套件、樣式庫、無頭 Headless UI
- 自動化文件開發與設計者會需要適應一直都在變化的網頁。
