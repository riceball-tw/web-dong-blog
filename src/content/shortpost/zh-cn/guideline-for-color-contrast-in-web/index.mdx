网页色彩的无障碍规范具体可以参考 WCAG 颜色对比度规范，通常会依照 WCAG 将色彩对比度划分为 AA 或 AAA 等级，更高强度通常更显眼且对更多用户识别上更友善（可以在 Chrome 检色器、 lighthouse 或各大工具中找到此项检测标准）。

不过事情没有这么简单。虽然 WCAG 颜色对比度是业界常用标准，但仍诞生在 CRT 显示器流行的时代，有很多原因导致它不再适合作为当今标准，[APCA](https://git.apcacontrast.com/documentation/APCAeasyIntro) 算法因而诞生并有望成为未来 WCAG 3 的新标准。

如果在这个时期希望有个折衷方案可以考虑看看 [Bridge-PCA](https://bridgepca.com/)，100% 兼容 WCAG2 对比度并解决现有问题。

依照色彩对比度进行严格的通过或不通过判定，作为指导方针并不具有指导性，而且不一定能解决特定使用者的需求。事实上，当谈到对比度时，不同使用者的需求是相互冲突的 —— 对某些人有益的，可能对另一些人有害。这一点甚至适用于字体大小。 - [APCA in a Nutshell](https://git.apcacontrast.com/documentation/APCA_in_a_Nutshell#failing-passfail)
