从很早以前就大致知道浏览器开始推出 Web Components 相关 API 与标准，但一直没有机会在实战中使用这项技术。随着时间过去发现 Web Components 的应用范围越来越广，并且也可以看到各大框架套件相继支持使用 Web Components 引起我的兴趣。简单来说，面对当前复杂的前端需求像是：

- 宣告式且简洁高效的模板系统
- 响应式的状态与对应的跨组件通讯
- 服务器端渲染 Web Components 一样都没有 😅，并不会取代现有的解决方案，不过可以期待更多网页组件的实现方式会向浏览器原生 API 靠拢。

研究了一下并写成文章：[了解網頁元件化的未來：Web Components](https://www.webdong.dev/post/future-of-the-web-native-web-components/)
